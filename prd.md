# Bini Puzzle MVP PRD

## 1. 프로젝트 개요

Bini Puzzle은 3×3 숫자 슬라이딩 퍼즐을 맞추는 독립형 웹 미니게임이다.

플레이어는 빈 칸과 인접한 숫자 타일을 클릭해서 이동시키고, 숫자 타일을 1부터 8까지 순서대로 정렬해야 한다.

이 프로젝트는 `Vite`, `TypeScript`, `Pixi.js`를 사용한다.

## 2. MVP 목표

이번 MVP의 목표는 **플레이 가능한 가장 작은 슬라이딩 퍼즐 게임을 완성하는 것**이다.

MVP에서 집중할 기능은 아래와 같다.

* 3×3 퍼즐 보드 표시
* 숫자 타일 1~8 표시
* 빈 칸 1개 표시
* 인접한 타일 클릭 시 이동
* 이동 횟수 표시
* 퍼즐 완성 여부 확인
* 클리어 화면 표시
* 다시 시작

디자인, 이미지 퍼즐, 난이도 선택, 타이머, 랭킹은 MVP 범위에 포함하지 않는다.

## 3. 게임 규칙

* 게임은 3×3 보드로 진행한다.
* 타일은 숫자 1부터 8까지 사용한다.
* 나머지 한 칸은 빈 칸으로 둔다.
* 플레이어는 빈 칸과 상하좌우로 인접한 타일만 이동할 수 있다.
* 타일을 클릭하면 해당 타일과 빈 칸의 위치가 바뀐다.
* 빈 칸과 인접하지 않은 타일은 클릭해도 이동하지 않는다.
* 숫자가 1부터 8까지 순서대로 정렬되고, 마지막 칸이 빈 칸이면 클리어다.
* 클리어 후 다시 시작할 수 있다.

## 4. 화면 구성

### 4.1 시작 화면

표시 요소:

* 게임 제목
* 간단한 설명
* 시작 버튼

예시 문구:

```txt
Bini Puzzle
숫자 타일을 움직여 1부터 8까지 순서대로 맞춰보세요!
```

### 4.2 게임 화면

표시 요소:

* 3×3 퍼즐 보드
* 숫자 타일 1~8
* 빈 칸 1개
* 이동 횟수
* 다시 시작 버튼

예시:

```txt
Moves: 12

1  2  3
4  5  6
7     8
```

### 4.3 클리어 화면

표시 요소:

* 클리어 문구
* 최종 이동 횟수
* 다시 시작 버튼

예시 문구:

```txt
Clear!
Moves: 24
다시 시작
```

## 5. 게임 상태

게임 상태는 MVP 기준으로 3개만 사용한다.

```txt
ready       시작 전
playing     게임 진행 중
clear       퍼즐 완성
```

### ready

* 시작 화면을 보여준다.
* 시작 버튼을 누르면 퍼즐을 섞고 `playing` 상태가 된다.

### playing

* 퍼즐 보드를 보여준다.
* 타일 클릭을 받을 수 있다.
* 이동 가능한 타일이면 빈 칸과 위치를 바꾼다.
* 이동 횟수를 증가시킨다.
* 이동 후 퍼즐이 완성되었는지 확인한다.

### clear

* 클리어 화면을 보여준다.
* 최종 이동 횟수를 보여준다.
* 다시 시작 버튼을 누르면 새 게임을 시작한다.

## 6. 주요 오브젝트

### 6.1 Board

3×3 퍼즐 상태를 관리한다.

역할:

* 현재 타일 배열 관리
* 빈 칸 위치 관리
* 타일 이동 가능 여부 확인
* 타일과 빈 칸 위치 교환
* 클리어 여부 확인

### 6.2 Tile

숫자 타일 하나를 의미한다.

역할:

* 숫자 표시
* 현재 위치 표시
* 클릭 이벤트 처리

### 6.3 Empty Cell

빈 칸을 의미한다.

역할:

* 타일이 이동할 수 있는 위치
* 숫자를 표시하지 않는다.

## 7. 이동 규칙

타일은 빈 칸과 상하좌우로 인접한 경우에만 이동할 수 있다.

인접 여부는 행과 열 기준으로 판단한다.

```txt
이동 가능:
타일과 빈 칸의 행 차이 + 열 차이 = 1

이동 불가:
타일과 빈 칸의 행 차이 + 열 차이 > 1
```

예시:

```txt
빈 칸이 (2, 1)에 있고
타일이 (2, 2)에 있으면 이동 가능

빈 칸이 (2, 1)에 있고
타일이 (0, 0)에 있으면 이동 불가
```

## 8. 클리어 조건

아래 배열이 되면 클리어로 판단한다.

```txt
1  2  3
4  5  6
7  8  빈칸
```

내부 데이터 예시:

```txt
[1, 2, 3, 4, 5, 6, 7, 8, 0]
```

`0`은 빈 칸을 의미한다.

## 9. 퍼즐 섞기 규칙

MVP에서는 반드시 풀 수 있는 퍼즐만 생성해야 한다.

가장 단순한 방식:

* 정답 상태에서 시작한다.
* 빈 칸을 기준으로 이동 가능한 타일을 여러 번 랜덤 이동한다.
* 이 방식으로 섞으면 항상 다시 풀 수 있는 퍼즐이 된다.

권장:

```txt
정답 배열에서 시작
→ 랜덤으로 30회 이동
→ playing 상태 시작
```

## 13. 성공 기준

MVP는 아래 조건을 만족하면 완료로 본다.

* `pnpm dev`로 게임이 실행된다.
* 시작 화면이 보인다.
* 시작 버튼을 누르면 3×3 퍼즐이 표시된다.
* 숫자 타일 1~8과 빈 칸 1개가 표시된다.
* 빈 칸과 인접한 타일을 클릭하면 이동한다.
* 빈 칸과 인접하지 않은 타일을 클릭하면 이동하지 않는다.
* 타일이 이동할 때마다 이동 횟수가 1 증가한다.
* 퍼즐이 `[1, 2, 3, 4, 5, 6, 7, 8, 0]` 상태가 되면 클리어된다.
* 클리어 화면에서 최종 이동 횟수를 볼 수 있다.
* 다시 시작 버튼으로 새 퍼즐을 시작할 수 있다.
* 콘솔에 치명적인 에러가 없다.
